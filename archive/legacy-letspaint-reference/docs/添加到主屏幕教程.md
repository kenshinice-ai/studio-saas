# 把 CMS 添加到主屏幕（变成 App 图标）

完成 PWA 改造后，管理后台和学员注册页都能像 App 一样添加到 iPhone / iPad / Mac 主屏幕——全屏运行、独立图标、无浏览器地址栏。零费用、无需上架。

## 前提

设备要能打开 CMS 网址（AWS 地址 `http://13.238.231.137`，或未来域名/HTTPS 地址）。**iPhone/iPad 上"添加到主屏幕"要求用 Safari 打开**（Chrome 不行）。

## iPhone / iPad（管理端，你自己用）

1. 用 **Safari** 打开管理后台地址并登录
2. 点底部中间的「分享」按钮（方框向上箭头）
3. 下滑选「**添加到主屏幕**」
4. 名称会自动显示「Paint CMS」，点右上「添加」
5. 主屏幕出现画板图标 🎨，点开即全屏运行，和原生 App 体验一致

> 提示：用隧道的 https 地址添加最稳定（局域网 IP 会随路由器变动）。登录状态会保留，PIN 锁照常生效。

## iPhone / iPad（学员注册页，发给学员）

学员侧同理：让学员用 Safari 打开 `你的地址/register`，按上面步骤添加。图标名称显示「Let's Paint」。这样家长查课时/看作品就像打开一个 App，不用每次记网址。

## Mac（Safari 16+）

1. Safari 打开管理后台地址
2. 菜单栏「文件」→「**添加到程序坞**」
3. 之后从程序坞点击即可独立窗口打开

## 常见问题

| 问题 | 解决 |
|---|---|
| 没有"添加到主屏幕"选项 | 必须用 Safari，不能用 Chrome/微信内置浏览器 |
| 图标是模糊的网页截图 | 确认 `icon-192.png`、`apple-touch-icon.png` 等已放进 CMS 目录并重启过服务 |
| 打开后还是有地址栏 | 删掉重加一次；确认 manifest.json 能访问（浏览器开 `/manifest.json` 应显示内容）|
| 换了网络/地址打不开 | 主屏图标记住的是添加时的网址；隧道地址变了需重新添加。建议用固定的局域网或域名 |

## 技术说明（了解即可）

- 新增文件：`manifest.json`（App 元信息）、`sw.js`（Service Worker）、三个 png 图标
- Service Worker 采用**网络优先策略**：除图标外一律实时请求服务器，**绝不缓存业务数据**——避免显示过期的课时余额
- 本质仍是你自托管的网页，数据依旧只在你的 Mac 上，不经过任何第三方
