# StudioSaaS 项目 Review 与实施清单

日期：2026-07-14

## 1. 逻辑与数据完整性

- [x] 把租户、订阅、报名状态迁移到统一状态机，拒绝任意跳转与不兼容组合。
- [x] `converted` 报名必须创建或关联真实学员，避免“已转化但无学员”。
- [x] 移除请求过程中执行数据库 DDL 的逻辑，结构变更只由 migration 管理。
- [x] 扩展租户归档快照，覆盖租户、成员、课程、课表、学员、课时、考勤、报名、媒体、作品集、通知、审计、订阅、用量、品牌草稿与版本。
- [x] 新租户创建先完成数据库事务，再生成工作区；文件失败时补偿删除数据库记录。
- [x] 租户编辑不再重新生成并覆盖门户工作区。
- [x] 工作区模板与元数据改为原子文件替换，避免半写入页面。

## 2. Super Admin 控制台

- [x] 保留并强化 MRR、付费租户、试用、即将到期、逾期、入驻与 30 天获客漏斗。
- [x] 商业风险提示改成可操作筛选按钮，可直接进入对应租户列表。
- [x] 普通租户编辑只处理资料、联系人、方案和日期；状态变更集中到带确认的审计动作。
- [x] Slug 在创建后只读，避免 URL、媒体路径和工作区路径断裂。
- [x] 方案配置由原始 JSON 提升为明确的报名、作品集、邮件模板、数据导出、优先支持权益开关，同时保留扩展标记。
- [x] 暂停、恢复、归档、还原和永久删除维持独立危险操作与确认短语。
- [x] 行业选择统一读取后端预设，不再在页面中维护另一套文案。
